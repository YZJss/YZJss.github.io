Linux防止ssh暴力破解

发表于 2019-02-27 | 分类于 main

Linux防暴力破解

最近服务器一直被人暴力破解…

lastb命令会显示最近错误登录的日志，包括通过ssh服务错误登录的日志。

没办法，只能修改登陆端口了。

1.先查看服务器端口范围

ssh端口范围在(32768~60999)之间就可以 ，也可以用netstat -lnp|grep 10086查看端口是否被占用

2.修改配置文件

vim /etc/ssh/sshd_config找到#Port 22修改成

3.重启服务

systemctl restart sshd.service，查看sshd状态systemctl status sshd.service

4.如果可以用新端口连接，把默认22端口禁用。

vim /etc/ssh/sshd_config在Port 22前加上#

ps:阿里云服务器需要在控制台添加安全组规则即可。

CentOS 7.6 安装MySQL教程

发表于 2019-02-25 | 分类于 main

CentOS 7.6 安装MySQL

下载MySQL的repo源

wget http://repo.mysql.com/mysql-community-release-el7-5.noarch.rpm

安装mysql-community-release-el7-5.noarch.rpm包

rpm -ivh mysql-community-release-el7-5.noarch.rpm

安装这个包后，会在/etc/yum.repos.d下获得两个MySQL的yum repo源：

安装MySQL

yum install mysql-server

安装完成后登陆MySQL并重置密码

mysql -u root

ps：如登录时有可能报这样的错：ERROR 2002 (HY000): Can’t connect to local MySQL server through socket ‘/var/lib/mysql/mysql.sock’ (2)，原因是/var/lib/mysql的访问权限问题。下面的命令把/var/lib/mysql的拥有者改为当前用户：

chown root /var/lib/mysql/

用 ll可 查看目录权限列表

更改文件拥有者 (chown )
chown 账号名称 文件或目录

重启MySQL

service mysqld restart

登陆，重置密码为123456

mysql -u root -p

重启生效 service mysqld restart

添加远程连接

mysql> GRANT ALL PRIVILEGES ON *.* TO root@"%" IDENTIFIED BY "password";

ps：阿里云服务器需要在控制台安全组规则中添加3306端口

配置my.cnf文件

vim /etc/my.cnf

在[mysqld]下添加bind-address = 0.0.0.0

:wq保存退出

如果找不到my.cnf文件执行mysql --help | grep my.cnf读取my.cnf配置位置的先后顺序

安装完成。
